Configurer le fichier `build.gradle`

Pour garantir que le projet Spring Boot fonctionne correctement sur WildFly, nous devons ajuster la configuration du fichier build.gradle. Voici les étapes pour exclure Tomcat (serveur intégré par défaut) et ajouter les dépendances spécifiques à WildFly.

Étapes

1. Exclure Tomcat

Spring Boot utilise Tomcat comme serveur web par défaut. Comme nous allons déployer notre application sur WildFly, nous devons l'exclure explicitement dans la configuration des dépendances.

Dans la section des dépendances du fichier build.gradle, ajoutez l'exclusion suivante pour Tomcat :

dependencies { implementation 'org.springframework.boot:spring-boot-starter-web' { exclude group: 'org.springframework.boot', module: 'spring-boot-starter-tomcat' } compileOnly 'jakarta.servlet:jakarta.servlet-api:6.0.0' providedRuntime 'org.wildfly:wildfly-dist:33.0.2.Final' testImplementation 'org.springframework.boot:spring-boot-starter-test' }

2. Ajouter les dépendances pour Jakarta Servlet et WildFly

Comme nous avons exclu Tomcat, nous devons ajouter les dépendances pour Jakarta Servlet (qui est utilisée dans les conteneurs Java EE) et indiquer que le runtime est géré par WildFly. Voici les dépendances à ajouter :

  • Jakarta Servlet API pour la compatibilité avec les standards Jakarta EE.

  • WildFly comme serveur d'exécution pour gérer le runtime.

dependencies { compileOnly 'jakarta.servlet:jakarta.servlet-api:6.0.0' // API de Jakarta EE providedRuntime 'org.wildfly:wildfly-dist:33.0.2.Final' // Serveur WildFly }

3. Générer le fichier WAR

Assurez-vous d'ajouter le plugin war dans la section des plugins pour générer un fichier WAR au lieu d'un fichier JAR:

plugins { id 'java' id 'war' // Plugin pour générer un fichier WAR id 'org.springframework.boot' version '3.3.4' id 'io.spring.dependency-management' version '1.1.6' }

Cela indique à Gradle de produire un fichier WAR lors du processus de build.

4. Commande de build

Une fois que tout est configuré, vous pouvez exécuter la commande suivante pour construire votre projet et générer le fichier WAR:

./gradlew build

Le fichier WAR sera généré dans le répertoire build/libs/ sous le nom monProjet.war.

5. Résultat attendu

À la fin de cette étape, votre projet Spring Boot sera configuré pour exclure Tomcat et utiliser WildFly comme serveur d'application. Le fichier WAR généré pourra être déployé directement sur WildFly via IntelliJ ou la ligne de commande.

Cette configuration permet de s'assurer que WildFly gère tout le cycle de vie du projet et que l'application est prête pour un déploiement en production.
