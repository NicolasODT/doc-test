Configurer Spring Boot pour générer des fichiers WAR

Dans cette étape, nous allons configurer Spring Boot pour générer des fichiers WAR au lieu de JAR, ce qui est nécessaire pour le déploiement sur WildFly.

Pourquoi utiliser des fichiers WAR au lieu de JAR ?

  • Les fichiers JAR incluent un serveur web intégré, comme Tomcat ou Jetty, ce qui n'est pas nécessaire lorsque vous déployez sur un serveur d'application comme WildFly.

  • Les fichiers WAR sont des archives web standard qui permettent à des serveurs comme WildFly de gérer les aspects Servlet et JSP.

Étapes

1. Modifier le packaging du projet en WAR

  • Ouvrez le fichier build.gradle et modifiez le type de packaging en WAR.

Voici un exemple de configuration dans Gradle:

plugins { id 'java' id 'war' id 'org.springframework.boot' version '3.3.4' id 'io.spring.dependency-management' version '1.1.6' } group = 'com.example' version = '0.0.1-SNAPSHOT' repositories { mavenCentral() } dependencies { // Ajoutez ici les dépendances nécessaires comme Jakarta EE et WildFly }

Dans cet exemple :

  • Le plugin war est utilisé pour générer un fichier WAR à partir du projet Spring Boot.

2. Ajouter une classe ServletInitializer

Pour permettre à WildFly de gérer le cycle de vie du projet Spring Boot, il faut ajouter une classe qui étend SpringBootServletInitializer.

Voici le code pour ajouter cette classe :

package com.example.demo; import org.springframework.boot.builder.SpringApplicationBuilder; import org.springframework.boot.web.servlet.support.SpringBootServletInitializer; public class ServletInitializer extends SpringBootServletInitializer { @Override protected SpringApplicationBuilder configure(SpringApplicationBuilder application) { return application.sources(DemoApplication.class); } }

Cette classe ServletInitializer permet à WildFly de gérer le déploiement et le démarrage de l'application Spring Boot comme un fichier WAR standard.

3. Générer le fichier WAR

Pour générer un fichier WAR, utilisez la commande suivante dans votre terminal :

./gradlew build

Le fichier WAR sera généré dans le dossier build/libs/ et sera prêt pour le déploiement sur WildFly.
